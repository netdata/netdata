# Flowtriq


<img src="https://netdata.cloud/img/flowtriq.png" width="150"/>


[Flowtriq](https://flowtriq.com/) is a DDoS detection and traffic analytics platform.
You can send notifications to Flowtriq using Netdata's Agent alert notification feature, which supports dozens of endpoints, user roles, and more.


<img src="https://img.shields.io/badge/maintained%20by-Netdata-%2300ab44" />

## Setup


### Prerequisites

#### 

- A Flowtriq account with a configured webhook endpoint
- Access to the terminal where Netdata Agent is running



### Configuration

#### Options

The following options can be defined for this notification

<details open><summary>Config Options</summary>



| Option | Description | Default | Required |
|:-----|:------------|:--------|:---------:|
| SEND_FLOWTRIQ | Set `SEND_FLOWTRIQ` to YES | YES | yes |
| FLOWTRIQ_WEBHOOK_URL | Set `FLOWTRIQ_WEBHOOK_URL` to your Flowtriq webhook endpoint URL. |  | yes |
| FLOWTRIQ_API_KEY | Set `FLOWTRIQ_API_KEY` to your Flowtriq API key. This is sent as an `X-API-Key` header for authentication. |  | no |


</details>



#### via File

The configuration file name for this integration is `health_alarm_notify.conf`.


You can edit the configuration file using the [`edit-config`](https://github.com/netdata/netdata/blob/master/docs/netdata-agent/configuration/README.md#edit-configuration-files) script from the
Netdata [config directory](https://github.com/netdata/netdata/blob/master/docs/netdata-agent/configuration/README.md#locate-your-config-directory).

```bash
cd /etc/netdata 2>/dev/null || cd /opt/netdata/etc/netdata
sudo ./edit-config health_alarm_notify.conf
```

##### Examples

###### Basic Configuration



```yaml
SEND_FLOWTRIQ="YES"
FLOWTRIQ_WEBHOOK_URL="https://app.flowtriq.com/api/webhooks/netdata"
FLOWTRIQ_API_KEY="your-api-key-here"

```


## Troubleshooting

### Test Notification

You can run the following command by hand, to test alerts configuration:

```bash
# become user netdata
sudo su -s /bin/bash netdata

# enable debugging info on the console
export NETDATA_ALARM_NOTIFY_DEBUG=1

# send test alarms to sysadmin
/usr/libexec/netdata/plugins.d/alarm-notify.sh test

# send test alarms to any role
/usr/libexec/netdata/plugins.d/alarm-notify.sh test "ROLE"
```

Note that this will test _all_ alert mechanisms for the selected role.
