# pushmark


<img src="https://netdata.cloud/img/pushmark.svg" width="150"/>


Send instant push notifications to your iPhone with our simple API designed for developers. Perfect for personal reminders, project alerts, and real-time updates that matter most to you.


<img src="https://img.shields.io/badge/maintained%20by-Netdata-%2300ab44" />

## Setup


### Configuration

#### Options

The following options can be defined for this notification

<details open><summary>Config Options</summary>



| Option                                                     | Description                                                                                                                                                                         | Default | Required |
|:-----------------------------------------------------------|:------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|:--------|:---------:|
| SEND_PUSHMARK                                              | Set `SEND_PUSHMARK` to YES                                                                                                                                                          | YES | yes |
| [DEFAULT_PUSHMARK_URL](#option-default-recipient-pushmark) | URL formed by the server-topic combination you want the alert notifications to be sent to. Unless hosting your own server, the server should always be set to https://pushmark.app. |  | yes |

<a id="option-default-recipient-pushmark"></a>
##### DEFAULT_PUSHMARK_URL

You can define multiple recipient URLs like this: `https://api.pushmark.app/KEY1` `https://api.pushmark.app/KEY2`

All roles will default to this variable if left unconfigured.

You can then have different servers and/or topics per role, by editing DEFAULT_PUSHMARK_URL with the server-topic combination you want, in the following entries at the bottom of the same file:
```text
role_recipients_pushmark[sysadmin]="https://api.pushmark.app/KEY1"
role_recipients_pushmark[domainadmin]="https://api.pushmark.app/KEY2"
role_recipients_pushmark[dba]="https://api.pushmark.app/KEY3"
role_recipients_pushmark[webmaster]="https://api.pushmark.app/KEY4"
role_recipients_pushmark[proxyadmin]="https://api.pushmark.app/KEY5"
role_recipients_pushmark[sitemgr]="https://api.pushmark.app/KEY6"
```

</details>



#### via File

The configuration file name for this integration is `health_alarm_notify.conf`.


You can edit the configuration file using the [`edit-config`](https://github.com/netdata/netdata/blob/master/docs/netdata-agent/configuration/README.md#edit-configuration-files) script from the
Netdata [config directory](https://github.com/netdata/netdata/blob/master/docs/netdata-agent/configuration/README.md#locate-your-config-directory).

```bash
cd /etc/netdata 2>/dev/null || cd /opt/netdata/etc/netdata
sudo ./edit-config health_alarm_notify.conf
```

## Troubleshooting

### Test Notification

You can run the following command by hand, to test alerts configuration:

```bash
# become user netdata
sudo su -s /bin/bash netdata

# enable debugging info on the console
export NETDATA_ALARM_NOTIFY_DEBUG=1

# send test alarms to sysadmin
/usr/libexec/netdata/plugins.d/alarm-notify.sh test

# send test alarms to any role
/usr/libexec/netdata/plugins.d/alarm-notify.sh test "ROLE"
```

Note that this will test _all_ alert mechanisms for the selected role.
