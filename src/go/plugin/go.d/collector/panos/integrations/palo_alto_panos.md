# Palo Alto Networks PAN-OS

Plugin: go.d.plugin
Module: panos

This collector monitors Palo Alto Networks PAN-OS firewalls through the PAN-OS XML API.

It collects read-only telemetry for BGP, system status, HA, environment sensors, licenses, and active IPsec SAs.

## Setup

Configure a job in `go.d/panos.conf`:

```yaml
jobs:
  - name: firewall
    url: https://192.0.2.1
    api_key: YOUR_PANOS_XML_API_KEY
    update_every: 60
    timeout: 10
    collect_bgp: yes
    collect_system: yes
    collect_ha: yes
    collect_environment: yes
    collect_licenses: yes
    collect_ipsec: yes
    max_bgp_peers: 512
    max_ipsec_tunnels: 1024
```

One of `api_key` or `username` plus `password` is required.

## Troubleshooting

- A metricset failure does not stop other metricsets from collecting. The error log includes the failing metricset and XML command context.
- Missing and malformed PAN-OS values are reported with their field and raw value when present instead of being converted to `0` or fake valid status.
- If PAN-OS reports an IPsec `<ntun>` count that differs from returned tunnel entries, the summary count is still emitted from `<ntun>` and the log explains that per-tunnel lifetime metrics may be incomplete.
- A `success response has no recognized telemetry payload` error means PAN-OS accepted the command, but the response did not contain the expected XML section for that metricset.
- Cardinality collection metrics report discovered, monitored, omitted_by_selector, and omitted_by_limit entities. If omitted_by_limit is non-zero, raise the matching `max_*` option or configure the matching selector.
