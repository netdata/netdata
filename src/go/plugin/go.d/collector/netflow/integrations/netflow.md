# NetFlow/IPFIX/sFlow


<img src="https://netdata.cloud/img/globe.svg" width="150"/>


Plugin: go.d.plugin
Module: netflow

<img src="https://img.shields.io/badge/maintained%20by-Netdata-%2300ab44" />

## Overview

Collects NetFlow v5/v9, IPFIX, and sFlow flow summaries over UDP.

The collector aggregates flow records into time buckets and exposes a flows Function response for Netdata Cloud aggregation.

This collector is supported on all platforms.

This collector supports collecting metrics from multiple instances of this integration, including remote instances.


### Default Behavior

#### Auto-Detection

This collector has no auto-detection. You must configure a job.

#### Limits

The default configuration retains a bounded number of buckets and keys to keep memory usage predictable.

#### Performance Impact

Flow processing is done in-memory and is designed to be lightweight. Higher flow rates or high cardinality can increase CPU and memory usage.

## Metrics

Metrics grouped by *scope*.

The scope defines the instance that the metric belongs to. An instance is uniquely identified by a set of labels.



### NetFlow/IPFIX/sFlow collector

These metrics refer to the local flow collector.

This scope has no labels.

Metrics:

| Metric | Dimensions | Unit |
|:------|:----------|:----|
| netflow.traffic | bytes | bytes/s |
| netflow.packets | packets | packets/s |
| netflow.flows | flows | flows/s |
| netflow.dropped | dropped | records/s |

## Setup


You can configure the **netflow** collector in two ways:

| Method                | Best for                                                                                 | How to                                                                                                                                 |
|-----------------------|------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| [**UI**](#via-ui)     | Fast setup without editing files                                                         | Go to **Nodes → Configure this node → Collectors → Jobs**, search for **netflow**, then click **+** to add a job. |
| [**File**](#via-file) | If you prefer configuring via file, or need to automate deployments (e.g., with Ansible) | Edit `go.d/netflow.conf` and add a job.                                                                        |

:::important

UI configuration requires paid Netdata Cloud plan.

:::


### Prerequisites

Configure your exporters (routers/switches/firewalls) to send NetFlow/IPFIX/sFlow to this Netdata node.

Ensure UDP traffic can reach the listener port (default `2055`).
